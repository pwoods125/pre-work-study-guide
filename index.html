HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Elements included within the head are not visible when a user looks at a webpage.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport
  • The charset (character set) element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.
  • The title element defines the title for a webpage and is required for Search Engine Optimization (SEO).
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The script element embeds client-side scripts in the HTML document.
  • Within HTML, “parent/child” language is often used to explain the relationship among elements.
  • Child elements are elements that are nested inside other elements.
  • The parent element is the element that holds all the child elements together.
  • The body element is known as a parent element.
  • The body element is an example of a semantic HTML element, where the name of the element describes its content and the role on a webpage.
  • Semantic HTML elements make HTML easier to read by other developers and browsers, and create accessibility to more users, including those who use assistive technologies such as screen readers.
  • Non-semantic elements exist, such as div, which creates a division in the page, and span that creates an inline division.
  • The h1 to h6 elements are headings, which are larger or more prominent titles and subtitles to help organize a page.
  • The h1 element is the biggest heading and should only appear once on each page.
  • The p element represents a paragraph or block of text.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items through the use of bullets or numbers.
  • All HTML elements can have attributes, which provide additional information about elements.
  • HTML attributes are always defined in the beginning element, never the closing element.
  • The img element contains information about images that are displayed on the webpage.
  • The img element can contain various attributes such as the src attribute, which defines the location or source of an image.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The a element is the anchor that creates links to the same or other webpages.
  • The href attribute within the a element points to a specific URL for a link.
  • The section elements are semantic elements that define a section in a document that contains thematic content.
  • HTML elements are case-sensitive.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property such as color: blue, which indicates the CSS property color and the value for that property, blue.
  • There are three ways to style a webpage using CSS, which includes inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline CSS is When a developer wants to style an element directly in the HTML file.
  • Each inline style must be edited individually whenever you make a design change.
  • Internal CSS style sheet embed an entire stylesheet directly in the HTML file using a style element in the head element.
  • External CSS style sheet allows developers to keep all their CSS rules in a separate file.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head.
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code, which can greatly reduce the overall size and complexity of a codebase.
  • The position property specifies the type of positioning method used for an element.
  • A link element creates the connection to the external file.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • A period preceding card designates this as a class selector, which allows any HTML element assigned to the class to gain the property assignments from the CSS.
  • The box-shadow CSS property adds shadow effects around an element's frame.
  • Assigning colors in CSS can be done in many ways, including by semantic references such as gray or blue, or by its hexadecimal code, or hex #.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout branch-name takes switches to that branch.
  • The git command must precede every Git statement.
  • git add -A : adds all modifications in the current working branch to the staging area, which informs Git that the changes will be added in the next commit.
  • git commit -m “DESCRIPTION OF CHANGES” :Commits the changes from git add -A, with specific description of what was fixed or changed.
  • git pull origin main : ensures the main branch is up to date and you are working with the correct patch.
  • git push origin branch-name : push the branch-name local changes to the remote GitHub branch.
  • Always push to the current working feature branch to avoid any possible bugs going into the main branch on accident.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is a programming language that allows users to interact with websites.
  • JavaScript is read from top to bottom, and the order of the code matters.
  • JavaScript is case sensitive.
  • Assignment operator is used to assign the specific data that we want stored in our variable.
  • A string is contained within quotation marks and refer to the variable's name.
  • Each line of JavaScript code ends with a semicolon.
  • The console.log method can be use to output a message to the web console by adding an argument inside the parentheses.
  • The word console within the console.log method refers to a test environment that developers use to check out their code.
  • The second part of the console.log method, .log(), is a method, which means a set of instructions that can be executed by a computer and will output whatever is added inside the parentheses to the console.
  • The script.js file is linked within the body element of the HTML.
  • To see console.log in action, we need to use Chrome DevTools.
  • The console.log method was created to make it easier for developers to test their code.
  • Using conditional statements such as if statements allow us to interrupt control flow.
  • An if statement is an example of a conditional statement and provides one or more conditions for the computer to evaluate.
  • When using the if statement, code will only execute if the statement meets a condition or is true.
  • If the condition is Falsy, or not truthy, the code will not execute, interrupting the control flow.
  • If one condition is truthy, the computer will execute the code that we provide between curly brackets.
  • We use else if when we want to provide more conditions than just if.
  • The strict equality operator (===) checks to see if two values are equal and returns a Boolean result true if the values are equal and false if the values are not equal.
  • An array is a single variable that is used to hold a group of data.
  • An index is a unique number locates the data item in the array.
  • Arrays are zero-indexed and sequential, meaning that the first item in any array has an index of 0, not 1.
  • The indices, or location identifiers, will always be whole numbers that increase by 1 each time.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • We can find the length of an array by using a built-in property of an array, appropriately named length.
  • A for loop contains the starting point for the loop, the condition, and a statement to allows the array to iterate over each item.
  • The code x++ is a shortcut way of writing x+1, so every time a code loops, x will increase by 1.
  • A for loop starts with for.
  • A commenting out code is used to make a block of code inactive without deleting it.
  • Functions are reusable blocks of code that contain instructions to perform a specific task.
  • To declare a function, we use the function keyword and then give our function a unique name followed by parentheses.
